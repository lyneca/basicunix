CD

In a UNIX system, you navigate through a directory structure from the command prompt. $ cd [path] The 'cd' command (short for 'change directory') moves the current working directory to the directory specified by 'path'.

Unix paths use forward slashes, and '~' signifies the home directory. ~ $ ls
Documents Desktop

~ $ cd Documents
~/Documents $ ls
document_1.txt document_2.txt

LS

'ls' lists the contents of the current working directory. $ ls 'ls' also has some options (known as 'flags').

To list all files (including hidden ones): $ ls -a To list all files and show attributes of the files (permissions, size, modified dates etc) $ ls -l These can be combined. $ ls -la

MV

'mv' moves a file to another location and name. $ mv file1 [path/]file2 Warning! If you overwrite a file with another file, the overwritten file will be permanently deleted!

CP

$ cp file1 [path/]file2 'cp' copies a file to another location and/or name.

Both 'mv' and 'cp' require the '-r' flag to act on directories. $ cp -r dir1 [path/]dir2

RM

'rm' removes files and directories. $ rm file Generally speaking, the UNIX command line has no 'recycle bin' like Windows and Mac. Be careful what you delete.

Some of the more useful 'rm' flags: $ rm -r file 'rm -r' deletes directories and all files inside.
'-r' stands for 'recursive'. $ rm -f file The '-f' flag forces the deletion, ignoring files that don't exist and never prompting before removal.
